Вот промпт, который нужно скопировать и отправить в нейросеть (например, ChatGPT с возможностью генерации кода, DeepSeek, Claude или GitHub Copilot).

Этот промпт сформулирован так, чтобы нейросеть выдала готовый, структурированный код для приложения (вероятно, на React/TypeScript), который реализует визуальное программирование нодами, экспорт в файл и инструкцию по интеграции в Unreal Engine.

---

Промпт:

Ты — senior full-stack разработчик и эксперт по интеграции веб-технологий с Unreal Engine 5.

Задача:
Создай код для отдельной вкладки (таба) в моем веб-приложении. Эта вкладка должна представлять собой визуальный редактор (Node-Based Editor), где я могу создавать логику с помощью нод (узлов), соединять их, а затем экспортировать результат в файл, который можно будет перенести в Unreal Engine.

Технические требования:

1. Структура приложения:
   · Нужен новый компонент NodeEditorTab.
   · Реализуй логику добавления новой вкладки в существующий интерфейс (например, рядом с существующими табами).
   · Используй современный стек: React, TypeScript, и библиотеку reactflow (или xyflow) для визуального редактирования нод.
2. Функционал нод:
   · Создай три типа нод (узлов):
     · Input Node: Позволяет задать имя переменной (string) и значение (float/vector).
     · Math Node: Выполняет операции (+, -, *, /) между двумя входящими значениями.
     · Output Node: Точка вывода результата. Имеет поле для задания имени выходного параметра.
   · Ноды должны иметь входные/выходные сокеты (handles) для соединения.
   · Должна быть возможность удалять ноды и соединения (через кнопку Delete или контекстное меню).
3. Экспорт в файл:
   · Создай кнопку "Export to Unreal".
   · При нажатии, скрипт должен парсить граф нод (все узлы и их связи) и генерировать JSON файл со следующей структурой:
     ```json
     {
       "version": "1.0",
       "nodes": [
         {
           "id": "node_1",
           "type": "input",
           "position": { "x": 100, "y": 100 },
           "data": { "name": "MyVar", "value": 10.0 }
         },
         {
           "id": "node_2",
           "type": "math",
           "data": { "operation": "add" }
         }
       ],
       "edges": [
         { "source": "node_1", "target": "node_2", "sourceHandle": "value", "targetHandle": "inputA" }
       ]
     }
     ```
   · Реализуй скачивание этого файла на диск пользователя с именем unreal_blueprint_graph.json.
4. Интеграция с Unreal Engine (инструкция):
   · После генерации файла, покажи пользователю модальное окно или тултип с инструкцией.
   · Инструкция должна содержать пример кода на Python (для скриптового редактора Unreal) или Blueprint объяснение, как прочитать этот JSON файл (используя ReadFile и ParseJSON) и создать на его основе простые узлы в Blueprint или Material Editor.

Требования к стилю и UX:

· Вкладка должна быть темной (Dark Theme), стилизованной под интерфейс Unreal Engine.
· Область с нодами должна занимать 100% высоты и ширины вкладки.
· Панель инструментов (Toolbar) должна содержать кнопки для добавления новых нод (Drag-and-Drop или кнопки с типами нод).

Ожидаемый результат:
Выдай мне полный код компонентов (NodeEditorTab, типы нод, логика экспорта), а также краткое описание того, как подключить эту вкладку к моему существующему приложению (если предположить, что у меня уже есть роутинг или таб-панель).

Почему этот промпт сработает эффективно:

1. Роль (Senior FS + Unreal Expert): Задает высокий уровень компетенции, чтобы нейросеть не писала "игрушечный" код, а учла серьезные требования к архитектуре.
2. Спецификация библиотеки: Указание reactflow (стандарт для нод в React) заставит нейросеть использовать правильные хуки (useNodesState, useEdgesState, Handle) вместо изобретения велосипеда.
3. Четкая структура JSON: Вы явно описываете, как должен выглядеть файл экспорта. Это важно, потому что "перенести в анрил" — абстрактно. JSON — это универсальный язык между вебом и движком.
4. Обратная связь (Инструкция для Unreal): Нейросеть часто "забывает", что пользователь не знает, что делать с файлом. Добавив требование выдать инструкцию с примером кода на Python для Unreal (или описание структуры для Blueprint), вы получаете законченный "продукт", а не просто редактор.
5. Детализация: Указание на "темную тему" и "удаление нод" делает результат более качественным и готовым к использованию.
